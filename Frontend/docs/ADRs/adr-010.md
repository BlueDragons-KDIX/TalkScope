# ADR-010: 操作（リモコン）ウィンドウの撤去とフロート操作UIへの移行

## ステータス
承認

## コンテキスト

従来、録音操作・発表終了・全リセットは `SystemControlWindow`（操作 / リモコンウィンドウ）に集約され、`attachSystemControlDock` によって全レイアウトの左端（または各プリセット内）に「削除不可ドック」として常時組み込まれていた。

この方式には次の難点があった。

- レイアウトツリーに削除不可ノードが混ざり、分割・リサイズ・追加/削除の各ロジックが `SYSTEM_CONTROL_WINDOW_ID` の特別扱いで複雑化していた。
- 操作 UI が常にレイアウトの一区画を占有し、画面を広く使いたい発表用途と相性が悪かった。
- 発表終了ボタンは操作ウィンドウ内にのみ存在し、表示/非表示の調整がしづらかった。

## 決定内容

### 1. 操作はレイアウト外の「フロート UI」に分離する

`FloatingControlDock` を新設し、**録音（開始/一時停止/再開）と全リセット**のみを担う。

- レイアウトツリーから独立した `position: fixed` のオーバーレイとして描画する。
- Pointer Events でドラッグ移動でき、ビューポート内にクランプ、リサイズにも追従する。
- 配色はアクセントテーマ（`--app-accent-rgb` / `useAccentTheme`）でボーダー・グローを与え、録音=緑 / 一時停止=橙のグラデーションで状態を明示する。サイズは操作しやすい中庸（録音ピル h-12、リセット 48px 円形）に固定する。

### 2. レイアウトからドック概念を撤去する

- `SystemControlWindow` の登録と実体を削除する。
- `attachSystemControlDock` / `tryUnwrapSystemControlDock` などドック専用ヘルパを撤去し、レイアウト生成・追加/削除ヘルパをドック非依存に整理する（`addWindowToLayout` / `removeWindowFromLayout` / `collectWindowIdsInLayout` / `findRightmostLeafWindowId`）。
- `ScriptableLayoutTemplates` のプリセットから `systemControl` リーフを除去し、比率を再構成する。

### 3. 発表終了は「機能温存・UI 非表示」とする

`PhaseTransitionButton` と `phaseStore.transitionTo` は残す。唯一の描画箇所だった操作ウィンドウを撤去することで UI 上は非表示化し、フェーズ遷移機能自体は将来再配置できる状態で温存する。

## 却下した選択肢

- **操作ウィンドウを残したまま発表終了ボタンだけ隠す**: ドックの特別扱いコストと画面占有の問題が残るため却下。
- **ドック関連ヘルパ・`systemControlWindow.ts` 定数を全削除**: `LayoutEngine` のリサイズ最小サイズ処理まで広範に書き換える必要があり回帰リスクが高い。発火しない防御コードとして残置し、変更面を限定した。

## 結果

- レイアウトツリーが純粋なユーザーウィンドウのみになり、分割・移動・追加/削除の見通しが改善する。
- 操作 UI が画面を占有せず、任意位置に移動できるようになる。
- マイク/モード選択は今回のフロート UI に含めず、既定値で動作する（必要時に別 UI で復帰可能）。
- 検証記録は `docs/tests/test-spec-018.md`。
