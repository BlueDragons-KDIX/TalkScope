# ADR-007: グローバルツールバーでのテスト／表示設定／全ウィンドウリセットとデモ重要語の責務分割

## ステータス
承認

## コンテキスト
リファクタ後の `presentation/App.tsx` は発表 UI のみで、従来 `app/App.tsx` にあったデモテキスト・ライブデモ・ダークモード設定・リセットが利用できなかった。またサーバー未接続の間も、将来の「重要語マーキング」を検証したいという要望があった。

`Frontend/AGENTS.md` には「デバッグコードは `debug/` に封じる」とある一方で、ツールバーから参照するシェル UI をどこに置くかが明文化されていなかった。

## 決定内容

### 1. 責務分割（`debug/` と `presentation/`）

| 置き場所 | 責務 |
|---------|------|
| `debug/demo/` | デモ用テキスト（`demo.ts`）、ストリーミングフック（`useDemoStream`）、サーバー代替のモック重要語リスト（`mockImportantTerms.ts`） |
| `presentation/` | ツールバー、`TestFeaturesPopover`、`DemoToolsProvider`、`useDemoImportantTermsSync` など「アプリの一部として見える」オーケストレーション |
| `stores/` | `demoImportantMarkingStore`（トグル）、既存ストアとの同期 |

モック重要語の ID は `demo-important-` プレフィックスで統一し、`termStore.stripDemoImportantTerms()` とサーバー由来語を区別できるようにする。

### 2. 文字起こしサービスへの外部入力

デモが音声認識と同じ表示経路を通るため、`ITranscriptionService` に `setTranscriptExternal(text)` を追加し、`WebSpeechTranscriptionService` で実装する。クリアは従来どおり `clearTranscript()`。

### 3. グローバルリセット

ウィンドウ内のリセットボタンをやめ、ツールバーの「リセット」から一括実行する。

- ライブデモ停止、録音停止、文字起こしクリア（サービス + `transcriptStore`）
- `termStore.resetSession()`（activeTerms / selectedTerm / searchHistory / pins / click weights）
- `bubbleStore.clearBubbles()`
- レイアウト（`layoutStore`）は変更しない

`TranscriptionView` は `showEmbeddedResetButton` でフッターリセットを抑制し、プレゼンレイヤの `TranscriptionWindow` では非表示にする。

### 4. 表示設定

ダークモード・アクセントカラーは既存の `SettingsModal`（`app/components/`）をモーダルで再利用し、`presentation/App.tsx` の状態と `document.documentElement.classList.toggle('dark', …)` で連動させる。

## 却下した選択肢

**デモ UI をすべて `debug/` 配下のパネルに閉じる**: ユーザー要求で「ウィンドウ外」「画面上部」と明示されたため、ツールバー側に配置した。

## 結果

- サーバー接続前でもモック語リストによりハイライト・バブルを検証できる。
- テスト用機能と本番フローの境界が ADR と AGENTS で説明可能になる。
- 将来サーバーから `termStore.addTerms` へ切り替える際、`useDemoImportantTermsSync` を差し替えるだけにしやすい。
