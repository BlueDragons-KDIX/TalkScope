# ADR-006: 文字起こし機能のメインシステムからの完全分離

## ステータス
承認

## コンテキスト
現在の `useSpeechRecognition` は App.tsx の状態管理と密結合しており、
`extractTerms`・`IT_TERMS`・類似度フィルタリングなどのフロント側の単語抽出ロジックと一体化している。

担当者から以下の要求があった。
- 「文字起こし機能はメインのシステムと全然関係がないので完全に切り離す」
- 「文字起こしのみで構成してください」
- 将来フロー: 文字起こし → サーバーへ送信 → サーバーで重要語抽出 → フロントへ反映（マーキング）

## 決定内容
- `ITranscriptionService` インターフェースを domain 層に定義
- `WebSpeechTranscriptionService` を infrastructure 層に実装
- `TranscriptionWindow` は `getTranscript()` でテキストを取得して表示するだけ
- フロントエンド側の単語抽出（`extractTerms`・`IT_TERMS`）を完全削除
- `transcriptStore` がテキストを保持し、他システムは store 経由で参照する

```
文字起こし（独立）
  ↓ transcriptStore に書き込む
重要語抽出以降（将来: サーバーから受け取る）
  ↑ transcriptStore からテキストを読む
```

## 却下した選択肢

**コールバックで渡す**: 現時点では不要。`transcriptStore` 経由で疎結合を実現できるため不採用。
将来必要になればコールバックを追加できる設計にしておく。

## 結果

- 文字起こしモジュールが完全に独立し、単独でテストできる
- フロントエンド側の単語抽出ロジックが消え、責務が明確になる
- 将来サーバー連携に切り替えるときも transcriptStore 以降のみ変更すればよい
