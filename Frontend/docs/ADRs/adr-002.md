# ADR-002: 状態管理に Zustand を採用

## ステータス
承認

## コンテキスト
現状は React hooks + App.tsx のみで状態を管理しており、
すべての状態が App.tsx に集中してコンポーネントが Props のバケツリレーになっている。
クリーンアーキテクチャへの移行に伴い、状態管理の分散が必要になった。

## 決定内容
Zustand を採用し、ドメインごとにストアを分割する。

```
stores/
├── phaseStore.ts      # 現在のフェーズ（発表中/発表後）
├── transcriptStore.ts # 文字起こしテキスト
├── bubbleStore.ts     # バブル状態・ライフサイクル
├── layoutStore.ts     # レイアウト状態
└── termStore.ts       # 用語・履歴・ピン留め
```

## 却下した選択肢

**React hooks + Context のみ（案A）**: 追加ライブラリ不要でシンプルだが、
状態が増えると App.tsx が再び肥大化する。Context は不要な再描画も起きやすい。
機能追加が予定されているため却下。

**Redux Toolkit**: 機能は豊富だが、このプロジェクトの規模にはオーバースペック。
ボイラープレートも多い。却下。

## 結果

- App.tsx が「フェーズの切り替えのみ担当」する薄いコンポーネントになる
- 各コンポーネントは必要なストアのみ購読し、不要な再描画を防げる
- ストアはクリーンアーキテクチャの application 層と相性よく連携できる
