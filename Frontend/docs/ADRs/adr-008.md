# ADR-008: プレゼンレイヤのヘッダー UI・フェーズ遷移ボタン・アクセント色の自動切替

## ステータス
承認

## コンテキスト
グローバルツールバー（ADR-007）に機能が集まり、狭い一行では整理しづらくなった。また「発表終了」と「発表中に戻る」で右側ボタン列の幅が変わり視覚的に不安定だった。発表後もレイアウト変更は意味上できないが、非表示にすると配置が跳ねる懸念があった。

アクセントカラーは設定で一貫していたが、発表後シーンを発表中と視覚的に区別したいという要望があった。

## 決定内容

### 1. ヘッダーのコンポーネント化と配置

- ツールバー UI は `presentation/components/PresentationAppHeader.tsx` に切り出し、`presentation/App.tsx` はフェーズ・アピアランス・リセット等の状態とコールバックのみを渡す。
- **開発者向けテスト**（`TestFeaturesPopover`）は `import.meta.env.DEV` のときのみ、左ブロック（フェーズバッジの右）に表示する。本番ビルドではバンドルから除去され、ヘッダー右の並びに影響しない。
- 右側は主にアイコンのみ（設定・リセット・レイアウト）。ラベルは `title` / `aria-label` で補う。

### 2. レイアウトプリセット（発表後）

- 発表後もレイアウトボタンは**表示したまま** `disabled` とし、スレート系の斜線オーバーレイで操作不可を示す（削除しない方針）。ドロップダウンは開かない。

### 3. フェーズ遷移ボタン

- **同一 DOM ボタン**で `during` / `after` を切り替える（削除・差し替えではなく色・文言・アイコン・ハンドラのみ変更）。
- 発表中: ローズ系の大きなピル型「発表終了」。
- 発表後: エメラルド系の同寸ピル型で「もどる」テキスト＋再生アイコン。
- 横幅・高さは固定（`w-[12.5rem]` 等）し、フェーズ切替で右側ツールバーがズレないようにする。

### 4. 発表後のアクセント色（設定色の「反対」）

- ユーザーが設定した `themeColor` は保存値として不変。
- **発表中**の `LayoutEngine` 等へ渡すアクセントは `appearance.themeColor` のまま。
- **発表後**は `presentation/utils/oppositeThemeColor.ts` の `getOppositeThemeColor()` でパレット上の補色に近い別キーへマッピングし、同じく `LayoutEngine` の `themeColor` として渡す。マップは `SettingsModal` の選択肢および `LayoutEngine` の `ACCENT_RGB` と整合させる。

## 却下した選択肢

**発表後にレイアウトボタンを DOM から除去する**: 右側のボタン数が変わり、他コントロールの位置が変わるため却下。

**発表後も設定色をそのまま使う**: フェーズ間の視覚的区切りが弱いため、補色マップによる自動切替を採用した。

## 結果

- ヘッダー責務がファイル単位で明確になり、本番でテスト入口を出さない方針をコードで表現できる。
- フェーズ遷移とレイアウト無効化の UX が文書化され、今後のツールバー拡張時の指針になる。
