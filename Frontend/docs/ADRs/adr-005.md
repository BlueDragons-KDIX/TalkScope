# ADR-005: フェーズシステム（発表中 / 発表後）の採用

## ステータス
承認

## コンテキスト
現在のアプリは「発表を聞いている最中」のみを想定している。
担当者から「発表終了後のサポート（議事録・重要語Top10等）も同じアプリで提供したい」と要望があった。

## 決定内容
`IPhase` インターフェースを domain 層に定義し、
発表中・発表後を独立したフェーズとして実装する。

```typescript
// domain/interfaces/IPhase.ts
interface IPhase {
  id: string
  name: string
  defaultLayout: Layout
  onEnter?: () => void
  onExit?: () => void
}
```

- フェーズはそれぞれ独立したレイアウトを持つ
- 発表終了ボタンで `during` → `after` へ遷移
- `phaseStore` が現在のフェーズを管理
- `App.tsx` は現在のフェーズに応じてシーンコンポーネントを切り替えるだけ

## 却下した選択肢

**タブやモーダルで切り替え**: フェーズ間でレイアウトが完全独立にならない。拡張性も低い。却下。

## 結果

- 発表後機能の追加が容易（新フェーズを実装して registry に登録するだけ）
- 各フェーズは独自のデフォルトレイアウトとウィンドウセットを持てる
- 現時点では発表後フェーズの中身は最小限で、今後拡張する
