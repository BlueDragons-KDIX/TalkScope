# ADR-004: ウィンドウ種別の拡張に IWindowDefinition インターフェース + レジストリを採用

## ステータス
承認

## コンテキスト
現在のレイアウトエンジンは `PanelId`（'transcription' | 'bubbleCloud' | 'detail' | 'history'）で
ウィンドウを識別しており、新種別の追加には types.ts と LayoutEngine.tsx の両方を修正する必要がある。
担当者から「ウィンドウの種類をこれから増やしていく予定」と明示された。

## 決定内容
`IWindowDefinition` インターフェースを domain 層に定義し、
`windows/registry.ts` に登録する方式に変更する。

```typescript
// domain/interfaces/IWindowDefinition.ts
interface IWindowDefinition {
  id: string
  label: string
  component: React.ComponentType<WindowProps>
  defaultSize?: { width: number; height: number }
}

// presentation/windows/registry.ts
// 登録・参照の窓口
```

新種別の追加はインターフェースを実装して registry に登録するだけ。
LayoutEngine は registry を参照してレンダリングする。

## 却下した選択肢

**switch 文で分岐**: 種別追加のたびに LayoutEngine を修正する必要がある。開放閉鎖原則に違反。却下。

## 結果

- LayoutEngine を変更せずに新ウィンドウ種別を追加できる
- 各ウィンドウは独立してテストできる
- フェーズごとに異なるウィンドウセットを持てる
