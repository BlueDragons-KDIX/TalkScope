# ADR-003: バブル重要度計算に Strategy パターン（DI）を採用

## ステータス
承認

## コンテキスト
現在のバブル重要度計算は「頻度 + クリック数 + コサイン類似度」が BubbleCloud コンポーネント内に一体化している。
担当者から「重要度のアルゴリズムは今後いろいろな方法で設定することになる」と明示された。

## 決定内容
`IImportanceStrategy` インターフェースを domain 層に定義し、
具体的なアルゴリズムを infrastructure 層の実装クラスとして分離する。

```typescript
// domain/interfaces/IImportanceStrategy.ts
interface IImportanceStrategy {
  calculate(term: Term, context: ImportanceContext): number
}

// infrastructure/importance/FrequencyStrategy.ts
// infrastructure/importance/VectorSimilarityStrategy.ts
```

BubbleCloud はインターフェース経由でのみ重要度を取得し、
アルゴリズムが変わっても BubbleCloud 側は変更不要。

## 却下した選択肢

**アルゴリズムを直接コンポーネントに書く**: アルゴリズム変更のたびに BubbleCloud を修正する必要がある。却下。

## 結果

- アルゴリズムの差し替えが BubbleCloud に影響しない
- 新しい重要度アルゴリズムを追加する際はインターフェースを実装するだけ
- 各アルゴリズムを独立してテストできる
