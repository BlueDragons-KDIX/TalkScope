# 作業記録: フロントエンドリファクタリング

**日付**: 2026-04-17

## 依頼された内容（プロンプト）
> AGENTS.mdを参照して、このプロジェクトのフロントエンドのみをリファクタリングして下さい。
> リモートブランチやこのブランチ以外のブランチに変更を加えないのであればプロジェクトの実行、確認のためのコマンドは基本的に自由に使ってもらって構いません。
> このままコミットするとコミット時のファイルが多すぎるのであとから見返しやすい単位でコミットしてください
> 構造が一度定まってからテストを追加するほうでおねがいします
> このブランチでは機能の追加変更はありません

## 実装・対応した内容

1. **全体構造の変更**
   `AGENTS.md` のアーキテクチャ構成に則り、`src/app` ディレクトリ内の中身を `domain/`、`infrastructure/`、`application/`、`presentation/`、`shared/` へ分割移動させました。
2. **依存パスの修復**
   ファイルの移動に伴い壊れたローカルインポート指定を絶対パス（`@/presentation/...`）等に修復するNodeスクリプトを記述して自動置換しました。
3. **TypeScript の検証**
   移動による不整合を防ぐため、`bunx tsc --noEmit`（最終的にはNode環境で `npm run build`）を行いエラーが出ない状態までコード修復を行いました。
4. **App.tsx の機能切り出し**
   UIのみに責任を持たせる Presentation 層の概念を厳密に守るため、`App.tsx` で抱えていたロジック（約600行分）を `application/hooks/useWorkspaceLayout.ts`、`useSessionTerms.ts`、`useTermVectors.ts` へ抽出し、可読性を大幅に向上させました。
5. **段階的コミットの作成**
   ファイル移動が巨大になるため、ドメイン層、インフラ層、アプリ層、プレゼン層ごとに論理的なコミットでGitの履歴に保存しました。
