# order-008: ウィンドウヘッダーの設定・閉じるボタンの操作性改善

## 指示要約

各ウィンドウヘッダー右側の **設定ボタン** と **閉じる（消去）ボタン** が小さく押しづらいため、次を実施する。

- ボタンとアイコンを少し大きくする
- アクセント色で視認性を強調する
- 閉じるボタンはアクセントの **補色側テーマ**（`getOppositeThemeColor`）を半透明で着色し、設定ボタンと区別する

---

## 実装方針

1. **`LayoutEngine` のヘッダー操作ボタンを共通化**
   - `WINDOW_HEADER_ACTION_BTN` で 32×32px のタップ領域・フォーカスリング・ホバーを統一。
   - アイコンは `16px`、`strokeWidth={2.25}`。

2. **設定ボタン**
   - 現在フェーズのアクセント RGB で半透明の背景・ボーダー・グローを付与。
   - 設定パネル表示中は `aria-pressed={true}`。

3. **閉じるボタン**
   - `getOppositeThemeColor(themeColor)` → `getAccentRgb` で得た補色側 RGB を半透明背景に使用。
   - `title="ウィンドウを閉じる"` を付与。

4. **スコープ外**
   - 設定ポップアップ内の閉じるボタン、レガシー `app/layout/LayoutEngine` は本オーダーでは変更しない。

---

## 完了条件

- 全ウィンドウヘッダーで設定・閉じるボタンが以前より大きく、色で識別できること
- 閉じるボタンが補色側アクセントの半透明着色であること
- ドラッグ用ヘッダー操作（`stopPropagation`）が従来どおり動作すること
- `npm run build` が成功すること
