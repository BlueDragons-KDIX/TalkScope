## テストの目的

order-008 に従い、ウィンドウヘッダーの設定・閉じるボタンのサイズと強調表示が改善され、レイアウト操作が破綻しないことを確認する。

## テスト項目

| # | テストケース | 種別 | 期待結果 |
|---|---|---|---|
| 1 | 設定ボタンサイズ | 手動確認 | 32×32px 程度の当たり判定でアイコンが 16px 程度に見える |
| 2 | 設定ボタン強調 | 手動確認 | アクセント色の半透明背景・ボーダー・グローが付く |
| 3 | 設定パネル連動 | 手動確認 | パネル表示中は `aria-pressed` が true になる |
| 4 | 閉じるボタンサイズ | 手動確認 | 設定ボタンと同サイズで押しやすい |
| 5 | 閉じるボタン色 | 手動確認 | 補色側アクセント（例: indigo→orange）の半透明着色で設定ボタンと区別できる |
| 6 | 閉じる動作 | 手動確認 | クリックでウィンドウがレイアウトから除去される |
| 7 | ヘッダードラッグ | 手動確認 | ボタン以外のヘッダーでウィンドウ移動（ドラッグ）ができる |
| 8 | フォーカス | 手動確認 | Tab 操作でフォーカスリングが表示される |
| 9 | 回帰（ビルド） | ビルド | `npm run build` が成功する |

## 手動確認手順

1. 発表中フェーズで複数ウィンドウを表示する。
2. 各ヘッダー右の歯車・×のサイズと色を確認する。
3. 設定を開き、再度歯車で閉じる。外側クリックでも閉じる。
4. ×でウィンドウを閉じ、ウィンドウ選択から再表示できることを確認する。
5. アクセント色を変更し、閉じるボタンの補色着色が追従することを確認する。
