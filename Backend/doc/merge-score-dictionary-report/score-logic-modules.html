スコア式の現状

現行の compute_term_score_additive() は以下の足し算で最終値を作る。 

base = count_axis_weight(occurrence_count, cap=100)
theme_linear = 0.5 * linear_cosine_similarity_to_unit(cosine(term_vector, theme_vector))
idf_scaled = 0.08 * idf_table.lookup(lemma)
final = max(0.0, base + sum(buffs))

occurrence_count が省略された場合は base=0。 THEME_EMA_ENABLED=False の場合はテーマベクトルを使わない。 idf_table が None の場合はIDFバフを付けない。

既存機能への接続候補

  • term_vector は既存辞書DBの Dictionary.meaning_vector または InfoTable.meaning_vector と接続できそう。 ただし次元や意味の粒度を揃える必要がある。
  • morph_tokens_for_scoring() は既存 analysis の形態素解析結果と接続できる。 ただし動詞・形容詞を含める既存TF-IDFとはフィルタ方針が違う。
  • IdfLookupTable は、将来 dictionary_terms.idf_wiki を使うなら変換レイヤを用意できる。 現状は新規 term_idf テーブルが正として設計されている。
